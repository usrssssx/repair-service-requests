# Заявки в ремонтную службу

## Запуск (Docker, предпочтительно)
1) `docker compose up --build`

Откройте `http://localhost:8000` и войдите через `/login`.

Важно: запуск через Docker выполняет `migrate --force`. Сиды запускаются только если таблица `users` пустая.

## Запуск (без Docker)
Требуется PHP 8.4+ (иначе `composer install` не установит зависимости).
1) `composer install`
2) `php artisan migrate:fresh --seed`
3) `php artisan serve`

Откройте `http://127.0.0.1:8000` и войдите через `/login`.

## Тестовые пользователи (из сидов)
- Диспетчер: `dispatcher@example.com` / `password`
- Мастер 1: `master1@example.com` / `password`
- Мастер 2: `master2@example.com` / `password`

## История действий
- В системе ведётся история событий по заявкам (создание, назначение, отмена, take, завершение).

## Проверка гонки
Вариант 1 (2 терминала):
- Войдите мастером, отправьте два запроса `take` на одну и ту же заявку (ожидается 200/302 и 409).

Вариант 2 (скрипт):
- Рекомендованные значения после сидов: мастер `id=2` (master1@example.com), заявка `id=2` (assigned).
- `MASTER_ID=2 ./race_test.sh 2`
- При необходимости: `MASTER_ID=2 BASE_URL=http://127.0.0.1:8000 ./race_test.sh 2`

Подсказка: после `migrate:fresh --seed` заявка со статусом `assigned` привязана к мастеру #2.

## Скриншоты (3 шт.)
- Форма создания заявки (`/requests/create`)
- Панель диспетчера (`/dispatcher/requests`)
- Панель мастера (`/master/requests`)

## Деплой (опционально)
- Подготовлена заготовка для Render: `render.yaml` (нужен аккаунт Render).
- Запуск через Docker (Render использует `Dockerfile`), переменные можно оставить по умолчанию.
- При необходимости задайте `APP_KEY` в настройках Render.
- На бесплатном плане Render сервис засыпает — первый запрос после простоя может быть медленным.

Альтернатива: `Procfile` для Heroku-совместимого запуска.
