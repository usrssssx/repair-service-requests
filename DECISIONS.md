# Архитектурные решения

- Laravel монолит + Blade (SSR): минимум инфраструктуры и быстрый старт, форм‑ориентированный UI без SPA.
- SQLite по умолчанию: не требует внешней БД, достаточно для тестового; миграции/сиды обеспечивают воспроизводимость.
- Простая сессионная авторизация через выбор пользователя из сидов: соответствует требованиям и исключает сложную аутентификацию.
- Доменная модель `RepairRequest` + enum `RequestStatus`: единый источник статусов и отсутствие конфликта с `Illuminate\Http\Request`.
- Сервисный слой `RepairRequestService`: концентрирует бизнес‑логику и логирование событий, контроллеры остаются тонкими.
- Безопасный `take`: атомарный `UPDATE` с условиями по статусу и исполнителю, при конфликте возвращается 409.
- История действий `RequestEvent`: фиксирует ключевые переходы и отображается в UI для прозрачности.
